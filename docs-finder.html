Turn current online members into playable teams.

This guide explains how Finder uses recent Community online data, player preferences and Looking for team status to suggest compatible groups without making another Tibia request for every match.

Panel setup

Publish Finder after online data exists

/finder set channel:#finder

The command stores one text channel and publishes the persistent Finder panel. Luminox needs View Channel, Send Messages, Embed Links, Read Message History and Manage Channels if the channel-name counter should update.

Recommended order

Configure the guild, world, Registration and Guild Members Online first. Finder can then reuse verified characters and current online snapshots instead of starting a second collection path.

Public presentation

Three numbers explain the current state

OnlineEligible characters in the most recent online check already collected by Luminox.
Looking for teamAccounts that intentionally published a temporary team-search message.
MatchesCompatible groups generated from the currently eligible characters.
Last updateThe exact refresh time plus Discord's relative time display.

Looking for team appears at both the beginning and end of a long Finder result so members can find the action without scrolling through every match. Long match lists split before Discord truncates them and retain navigation back to the first page.

Match score

Compatibility, not random combinations

Shared XPCharacters must fit Tibia's shared-experience level range.
VocationsBalanced parties score better without forcing every duo into one template.
ActivityOnline duration, session levels and rolling daily levels influence relevance.
PresentationEvery character receives one compact line with a clickable profile.

The percentage is a recommendation score, not a promise that every suggested member wants the same hunt. Looking for team gives members the explicit signal needed to turn a technically compatible group into a real conversation.

Group rules

Duos stay flexible; larger parties stay viable

  • Duos may combine any two compatible vocations, including two DPS characters.
  • Groups of three, four or five require at least one Elite Knight and one Druid.
  • Duplicate or clearly inferior combinations are avoided.
  • Matches are ordered by score and split before Discord message limits.

Looking for team

Advertise even while offline

  1. Click Look for team.
  2. Select a registered character or choose to reuse the main character.
  3. Optionally describe the desired team; otherwise Luminox shows Any team.
  4. Select one to five hours.
  5. Review the created and expiry timestamps in the public list.
  6. Use the same control to leave the list early.

The signal belongs to the Discord account, uses an account-wide escalating anti-spam cooldown and is deleted after expiry. Switching characters or changing the description does not create a separate cooldown identity. The signal appears only in Finder and can remain visible while the character is offline so other members can contact the player through Discord.

Match history

Private, paginated and clickable

Match history returns recent generated groups privately, preserving the one-character-per-line layout and profile links from the public panel. Page buttons reuse one private response instead of creating a new message for every page.

Only recent history is kept. It helps members recover a suggestion but is not a permanent activity archive and does not award Loyalty Points.

Performance

Reuse online data at the selected service speed

Finder is available from Community Growth. The plan controls how often Luminox may request fresh external game data: up to every five minutes on Growth, every two minutes on Scale and every minute on Founder.

The public Finder panel can refresh from cached information between those checks. It reuses the latest compatible online result instead of starting another Tibia scan for every redraw. Expired Looking for team messages and old match history are cleaned automatically.

  • No separate online-page request for every Finder refresh.
  • No duplicate character profile created for match calculation.
  • No permanent storage for expired Looking for team descriptions.
  • No matchmaking result changes guild roles, event teams or character registration.
