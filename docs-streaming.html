Administrator setup

Enable only the voice channels intended for streaming

/loyalty streams channel:Stream enabled:true

Run the command once for every voice channel where members may earn Stream Loyalty. Use the same command with enabled:false to remove a channel later. The configured channels appear in the server configuration overview; members do not need a separate command or panel.

Recommended permissions

Members need permission to view and connect to the voice channel and to stream their screen. They do not need permission to speak if the channel is designed as a silent streaming room.

Eligibility

The account must be a verified, active guild member

  • The Discord account needs at least one verified character in the configured guild and world.
  • The member must be connected to an enabled voice channel.
  • Discord must report that the member is actively sharing their screen.
  • The session must last at least ten minutes before it becomes rewardable.
  • Blacklist, Ban List and normal Loyalty eligibility rules still apply.
  • Simply sitting in voice, speaking or watching another stream does not count.

Reward calculation

Ten points per hour, with every valid minute preserved

Luminox calculates the reward proportionally instead of discarding incomplete hours. A valid 90-minute session contributes the equivalent of 15 Loyalty Points. Fractional progress is retained internally, so several shorter valid sessions can eventually produce the next whole point.

Session endThe final duration and reward are settled once, preventing repeated grants.

Abuse protection

A rolling daily limit keeps the reward healthy

Each account can receive rewards for up to six streamed hours in a rolling 24-hour window. The limit follows the account, not an individual character or voice channel. Moving between enabled channels therefore cannot reset it.

Below-minimum sessions

A session shorter than ten minutes is closed without Loyalty Points. It does not become a reusable fragment that can be spammed repeatedly.

Member visibility

Streaming progress appears where members already check Loyalty

The private View my Loyalty Profile response includes current streaming status, total valid streamed time, rewarded streamed time, points already earned, pending progress and rolling 24-hour usage.

The main Leaderboards panel also includes Total Streamed Hours. This ranking uses the same stored session totals and does not trigger extra Discord or Tibia requests.

Reliability

Discord events plus regular safety saves

Luminox reacts when Discord reports a member starting, stopping, joining, leaving or moving a stream. It saves active progress every ten minutes so a restart does not discard a long session. Each finished stream has one stable record, which prevents it from being rewarded twice.

  • No additional Tibia requests are required.
  • No per-second database writes are performed.
  • Active state is restored safely after normal bot restarts.
  • Leaving the guild or losing Loyalty eligibility removes stream progression with the rest of the account profile.
