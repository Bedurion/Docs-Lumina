No manual drift

The identity follows the member's verified account.

Nickname updates reuse the same trusted registration and character information that already powers roles, panels, Loyalty and guild membership checks.

Main-driven identity

The selected main represents the account. Changing main updates the public Discord identity instead of leaving the previous character behind.

Current vocation and level

Scheduled character synchronization refreshes vocation and level from Tibia data according to the server's available refresh policy.

Account-wide Loyalty

The L value represents the Discord account's Loyalty Level, so alternate characters do not split the member's progression.

Name-change continuity

When the Tracker confirms a character rename, the stored identity follows the same character and the next nickname refresh uses its new name.

Safe guild changes

If a main is no longer eligible but another registered guild character remains, Luminox can promote the highest-level eligible character.

Honest permission handling

If Discord role hierarchy blocks an edit, Luminox reports the suggested nickname rather than pretending that synchronization succeeded.

What keeps it current?

Important identity changes trigger a refresh.

  1. 01
    Registration succeeds

    The first verified character becomes main and establishes the account identity.

  2. 02
    Main or character data changes

    Main selection, vocation, level and confirmed renames flow into the nickname.

  3. 03
    Loyalty changes

    Rewards, reversals and manual adjustments recalculate the displayed Loyalty Level.

  4. 04
    The member requests an update

    /update refreshes that account without launching a full-server scan.
