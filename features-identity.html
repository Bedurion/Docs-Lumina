The problem

Discord names alone are not enough for a serious guild.

Members rename accounts, use alternate characters and move between ranks. Manual verification quickly creates stale nicknames, incorrect access and uncertainty about who owns a character.

Luminox verifies ownership through a temporary code in the Tibia.com character comment. The configured guild and world establish the server context, while the selected main character becomes the member's public identity.

What changes after verification

One verified account can power the whole server.

Automatic role mapping

Map Tibia guild ranks to Discord roles. Guests outside the guild can receive a separate Guest role without inheriting guild access.

Readable nicknames

Keep character, vocation, level and Loyalty level visible in a compact format such as Name [RP360 L8].

Account-level history

Multiple characters can belong to one Discord account while Loyalty, GuildBank and staff activity remain attached to the account.

Whois visibility

Authorized guild members can identify the registered owner and see that account's verified characters without exposing unverified identities.

Safe main changes

Members can change their main from the Registration panel. If a main leaves the guild, the highest eligible registered character can take over.

Restriction integration

Blacklist and Ban List restrictions remove managed guild access and Loyalty eligibility without erasing the verified registration history.
