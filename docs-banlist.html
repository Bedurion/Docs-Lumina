Documentation · Ban List

Configure clear, timed and reversible sanctions.

Ban List is available in Universal and Community Edition. Administrators publish one permanent panel; authorized staff then add, edit and lift sanctions through guided Discord controls.

Luminox record, not a hidden Discord command

The panel maintains Luminox's sanction and connected access state. It does not claim to replace Discord's native server-ban interface unless your own server process also applies that action.

Panel setup

Publish the panel once

/banlist set channel:#ban-list

The command stores the destination and publishes or refreshes the persistent Ban List panel. Use /banlist overview to inspect the current channel without republishing it.

Panel destination/banlist set
Configuration check/banlist overview
Moderation log/moderation set channel:#log

Add ban

Choose the target, reason and duration

  1. Press Add ban. Luminox opens a private target selector for the responsible staff member.
  2. Select a Discord member. This route is available in both editions and avoids typing an account manually.
  3. Community alternative. Choose Add Tibia character when the exact character is the relevant identity.
  4. Explain the reason. The reason is required and appears in the readable panel record.
  5. Choose duration. Enter a number of days or use unlimited when no automatic end date applies.
No CRUD command clutter

Daily additions happen through the panel. Slash commands remain focused on administrator setup and overview.

Community Edition

Registered Tibia identity adds context

When a target is connected to registered Community data, Luminox can retain character information and reconcile managed guild access across the linked Discord account.

  • Registered ownership is preserved rather than deleted.
  • Managed guild roles and Loyalty eligibility can be removed while the sanction is active.
  • Unrelated Discord roles are not treated as guild roles and remain outside this reconciliation.
  • All linked registered characters are considered so an alternate character cannot silently restore managed access.
  • Lifting the sanction allows normal synchronization to evaluate the account again.

Edit or lift

Manage the existing record instead of creating a duplicate

  1. Press Edit or lift ban.
  2. Select the target from the paginated manager.
  3. Choose Edit reason, Set duration, Set unlimited or Lift ban.
  4. Confirm the result in the refreshed panel and moderation log.

Existing character-based records remain compatible after the Discord-user target flow is enabled because the manager uses a stable record key rather than the visible label alone.

Long lists

Selectors and public output remain bounded

  • The manager shows up to 25 targets per selector page.
  • Previous and Next controls appear only when another page exists.
  • The public panel can split long output across multiple Discord messages.
  • Navigation remains attached to the final live panel rather than duplicating controls on every message.
