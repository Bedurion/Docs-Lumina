Before it can work

No separate nickname panel or setup command is required.

Automatic Tibia nicknames are a Community Edition identity function. They become available after the guild and world are configured, the Registration panel is published and a member verifies at least one character.

EditionCommunity Edition.
Identity sourceA verified character selected as the Discord account's main.
Bot permissionManage Nicknames in Discord.
Role hierarchyThe Luminox role must be above the member's highest editable role.
Foundation/guild set, rank mapping where needed and /registration set channel:#registration.
Member-facing workflow

Members register and manage characters through the permanent Registration panel. Nickname synchronization happens as a result of that verified identity; members do not need to memorize a nickname command.

Exact output

Read Sacrel [RP394 L9] at a glance.

Sacrel [RP394 L9]
SacrelThe current tracked name of the selected main character.
RPThe compact vocation code. Here it means Royal Paladin.
394The latest stored Tibia level for that main character.
L9The Loyalty Level of the Discord account, shared by all its registered characters.

There are no spaces between vocation and character level, or between L and Loyalty Level. The compact format preserves room for long Tibia names while remaining readable in member lists and messages.

The character name, vocation and level come from the main character's stored Tibia snapshot. Loyalty Level comes from the verified Discord account rather than from one character.

Synchronization

Important changes refresh the nickname automatically.

  • A successful first registration establishes the main and applies its identity.
  • Changing the main character rebuilds the nickname from the newly selected main.
  • Scheduled character synchronization applies changed name, vocation or level data.
  • Loyalty rewards, reversals and manual adjustments refresh the displayed Loyalty Level.
  • /update lets a member refresh their own current main, managed role and nickname without scanning every account.

These are supported refresh policies, not a promise that Tibia.com exposes a changed value instantly. Luminox avoids extra scans when current stored data can be reused safely.

Main-character rules

The nickname represents the account, not every alternate.

The first verified character becomes main automatically. A member with several registrations can choose another main from the Registration panel. The nickname then follows that selection.

If the main stops belonging to the configured guild while another registered character still belongs, Luminox can promote the highest-level eligible guild character and record the identity change. Confirmed Tibia name changes preserve the same stored registration and update the nickname instead of creating a second character.

Registration remains the source of truth

Changing a Discord display name manually does not change the verified main, Tibia snapshot or Loyalty account. Luminox rebuilds managed identity from verified data during the next eligible refresh.

Discord boundaries

Luminox never claims an edit that Discord rejected.

Server ownerDiscord does not allow a bot to change the server owner's nickname.
Higher roleA member whose highest role is above Luminox cannot be edited.
Missing permissionWithout Manage Nicknames, automatic nickname updates cannot be applied.
FallbackLuminox can return the exact suggested nickname so an authorized person can apply it manually.
Unrelated rolesNickname synchronization does not remove unrelated Discord roles.
