Configure automatic nicknames for either edition.

Universal Edition uses one member-provided game name and Loyalty Level. Community Edition verifies a Tibia main and adds its vocation, current level and Loyalty Level. This guide covers both paths without mixing their requirements.

Start with the correct path

Both editions create nicknames. Their identity sources differ.

The server edition is selected during installation. Luminox automatically publishes the matching member workflow, so administrators do not maintain two panels or ask members to choose a technical mode.

Universal EditionThe member provides one game name for this Discord server. No external game website is contacted.
Community EditionThe member verifies Tibia character ownership and selects one registered character as main.

Panel-first by design

Members do not configure nicknames with slash commands. Staff publish one permanent panel; its buttons become Set game name and My profile in Universal, or Register character, My characters and Change main character in Community.

Administrator setup

One command publishes the correct panel.

/registration set channel:#profilesAdministrator
What it is

The shared setup command for Universal member profiles and Community character registration.

What it does

Saves the text channel and creates or updates one permanent, edition-aware panel.

How it works

Luminox reads the server edition and publishes only the controls supported by that edition. Reusing the command replaces or refreshes the existing panel instead of creating a second workflow.

/registration refreshAdministrator
What it is

A safe manual refresh for the already configured panel.

What it does

Rebuilds the panel copy and buttons from the current edition without changing stored member profiles.

How it works

Use it after changing edition or when the Discord message needs to be republished. It does not run a Tibia character scan.

Community and Founder servers must also configure their Tibia guild and world before members can complete verified character registration. Universal servers do not need /guild set for member nicknames.

Universal Edition

A lightweight profile for any game or community.

  1. 01
    Click Set game name

    The member opens the permanent Member Profile panel and enters the name they use in their game.

  2. 02
    Luminox validates it locally

    The name must contain 2–24 visible characters, cannot contain square brackets and must be unique inside that Discord server.

  3. 03
    The profile is stored per server

    The same Discord account can use a different game identity in another server without the profiles colliding.

  4. 04
    The nickname becomes Game Name [L#]

    No external game API, vocation, character level, alternate-character list or main-character selection is used.

Member-provided means intentionally simple

Universal Edition does not claim that the game name is externally verified. It gives general gaming communities a consistent profile and Loyalty identity without tying Luminox to one game.

Community and Founder Edition

Verified Tibia identity adds live character context.

The first verified character becomes main. Members can register alternates and change their main through the panel. Luminox then builds the nickname from the current tracked main.

The character name, vocation and level come from the main character's stored Tibia snapshot. Loyalty Level belongs to the verified Discord account, so alternates do not split progression.

If the main leaves the configured guild while another registered character remains eligible, Luminox can promote the highest-level eligible character. Confirmed Tibia name changes preserve the same registration and flow into the next nickname refresh.

Discord boundaries

Luminox never claims an edit that Discord rejected.

Server ownerDiscord does not allow a bot to change the server owner's nickname.
Higher roleA member whose highest role is above Luminox cannot be edited.
Missing permissionWithout Manage Nicknames, automatic nickname updates cannot be applied.
FallbackLuminox can return the exact suggested nickname so an authorized person can apply it manually.
Unrelated rolesNickname synchronization does not remove unrelated Discord roles.
