Bot · Persistent identity

Keep tracking a character after its name or world changes.

Luminox follows stored Tibia characters through renames, trades, transfers, returns and deletion. Blacklist and Ban List history therefore remains attached to the same character.

In plain English

Names can change. The record should not disappear.

The tracker periodically checks risk characters already stored in Blacklist or Ban List. It detects supported identity changes, updates connected panels and creates a permanent Discord log when something meaningful changes.

A name change alone is not a defeat. A trade can be partially defeated while the character remains on the configured world. A transfer away can become completely defeated, and a later return is detected too.

Supported outcomes

Each change means something different.

Changed

The new name replaces the current display while preserving the old identity connection.

Traded

The record is marked traded; it remains active if the character still belongs to the configured world.

Transferred

The character moves into the Out of this world section and leaves active enemy lists.

Returned or deleted

Returns restore active relevance; deleted characters or disbanded guilds move into resolved history.

Retry-safe alerts

Important notices are sent once, even after a temporary failure.

Luminox records whether a Guild Chat alert was delivered. A recent missed alert can be retried within a bounded window without repeatedly notifying the same change.
