Installation

Wizard plus permanent overview

/install wizard/install overview

The wizard detects missing configuration and offers only the next safe steps. It does not erase existing channels, guild verification, roles or data. Overview always shows the active profile and current configuration.

Controlled rollout

Build authority before publishing member actions

  1. Verify the guild and world. This establishes the identity boundary for Registration and guild membership.
  2. Map ranks and Guest. Place the Luminox role above every role it must manage.
  3. Configure global moderators. Add every legitimate moderator role, then define specialist roles for protected modules.
  4. Publish Registration. Test verification, timezone, main selection and role synchronization with a controlled account.
  5. Add member panels. Events, Finder, Loot and Loyalty become useful once identity is reliable.
  6. Add protected operations. GuildBank, Recruitment, Guildhall, Support and moderation should be tested with both authorized and unauthorized accounts.

Discord permissions

Administrator is a Discord permission

Luminox does not maintain a second administrator list. A Discord member with the Administrator permission receives administrator access. Place the bot role above every rank, Guest, Alerts and specialist role it must manage.

Role hierarchy wins

Even with the correct permission checkbox, Discord blocks changes to members or roles positioned above the bot. The server owner nickname is never editable by a bot.

Staff access

Multiple global moderator roles

/moderators roles add role:@Vice Leader/moderators roles overview

Specialist actions add a second requirement: Treasurer for GuildBank review, Recruitment for recruitment review, Guildhall Staff for room management and category roles for Support. Administrators bypass specialist-role requirements.

Cleanup and history

Keep useful records without storing everything forever

  • Automatic local backups run every 24 hours, retain 30 daily copies and 12 monthly copies.
  • Support and event conversations remain archived Discord threads instead of being copied into another transcript store.
  • Old Loyalty detail, GuildBank review detail and temporary states follow documented cleanup periods.
  • /autodelete set supports immediate cleanup or timed deletion from ten minutes to thirty days; /autodelete disable removes either rule explicitly.

Launch checklist

Test one allowed and one denied path

  • Run /install overview and verify every configured channel and role mention.
  • Confirm a normal member cannot use administrator or specialist actions.
  • Confirm a moderator without the specialist role is denied where two-role access is expected.
  • Create one test registration, panel refresh, event and private ticket.
  • Verify permanent logs remain above live control panels.
  • Check mobile presentation and channel counters after the first automatic refresh.
  • Keep the bot process owner, environment credentials and backup directory outside normal Discord staff access.

Operations

Stop cleanly and keep publishing credentials private

Run the bot with the project start or development script and stop the active process with Control + C. Backup management commands are intentionally not published to every Discord server.

The optional Website publisher is connected privately by the deployment owner. Never paste repository credentials into Discord commands, panels, screenshots or data files. Read the Website publishing guide →
