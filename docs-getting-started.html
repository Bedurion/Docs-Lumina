The operating model

Commands build the workspace. Panels run it.

Administrator configures

Use a setup command to choose the channel, roles and limits for one system.

Luminox publishes

The bot posts a persistent panel with the correct controls and explanation.

Community operates

Members and staff click buttons, selectors and forms in Discord.

Discord preserves context

Live panels refresh while threads and permanent logs retain history.

Nothing extra to learn or host

There is no separate Luminox control website. This documentation explains the system, but configuration and daily operation stay inside your Discord server.

Members

Your first five minutes depend on the edition

Find the public panels

Open the channels your administrators configured for Registration, Support, Events and other enabled systems.

Click the relevant action

Buttons and selectors open the next guided step privately when personal input is required.

Community: verify your character

Use Registration, place the temporary code in the Tibia comment and submit the same character again.

Universal: use Discord identity

Game-independent panels work with your Discord server account and do not require a Tibia registration.

Members do not need to memorize an operational command list. In Community Edition, /update remains as an optional targeted utility when your own main character, nickname or managed guild role needs an immediate refresh.

Staff

Know the difference between global and specialist access

  • A configured global moderator role enables common moderation workflows.
  • GuildBank review additionally requires Treasurer.
  • Recruitment review additionally requires the configured Recruitment role.
  • Support categories can require one of several category-specific staff roles.
  • Discord administrators retain administrative access and every action remains attributable.

Administrators

Use a safe configuration order

  1. /install wizard
  2. /guild set
  3. /moderators roles add
  4. /ranks map and /ranks guest
  5. Publish panels from the Panel Directory
  6. /install overview
Never rebuild a working server unnecessarily

The wizard detects missing configuration. Re-running it does not require deleting existing channels, panels or registrations.

Daily operation

Most systems maintain themselves

Immediate actions

Buttons, reviews and configuration changes update their relevant panel without waiting for a full external scan.

Plan-based refresh

Eligible external data uses 15-minute Free, 10-minute Core, 5-minute Growth, 2-minute Scale or 1-minute Founder policy.

Lightweight panels

Cached panels use 5-minute Free, 3-minute Core, 2-minute Growth and 1-minute Scale or Founder policy where supported.

Individual systems can refresh immediately after meaningful changes. Use manual refresh commands only when the automatic interval is not appropriate for the situation.

Help

Use the private Support panel

Open the configured Support channel, choose the correct category and answer every required question. The resulting thread is private to you and responsible staff.
