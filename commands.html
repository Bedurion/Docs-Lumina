Documentation · Commands

Every active command, explained individually.

Luminox currently registers 32 slash-command roots plus one message context command. This reference expands every available subcommand separately, so administrators can understand exactly what each action changes before running it.

Commands versus panel actions

Slash commands configure the server, inspect private settings or run deliberate maintenance. Repetitive member actions—registration, event creation, loot splitting and moderation-list management—use guided panel buttons instead, reducing command clutter and input mistakes.

Administrator commands normally use Discord's Administrator permission. Configured moderator and specialist roles can operate the relevant panel buttons without receiving unrestricted server administration.

Member tools

Available to everyone

/infoEveryone

Opens your private Luminox account overview. It summarizes registered characters, the selected main, Loyalty progression, GuildBank contribution and other account-linked data without exposing it publicly.

/updateEveryone

Refreshes your own main character immediately and reapplies the nickname and managed guild role derived from current data. Use it when your level, guild rank or main-character details changed before the next automatic synchronization.

/timezone setEveryone

Stores your personal timezone for event creation, scheduling and time conversion. The selected zone stays attached to your Discord account and is reused across every Luminox system.

/whois character:NameEveryone

Looks up a verified character and shows its registered Discord owner, account characters, Tibia information, Loyalty and GuildBank data. Unregistered or unverified characters return a limited result with a Tibia.com link.

Apps → Convert timeEveryone

Runs from Discord's message context menu on a message containing a time. Luminox compares the author's configured timezone with yours and posts a short conversion response. If your timezone is missing, the response links directly to /timezone set instead of assuming a default.

First configuration

Installation, guild identity and access

/install wizardAdministrator

Starts the non-destructive installation assistant. It detects missing configuration, preserves existing settings and directs the administrator to the exact command needed for each unfinished system.

/install overviewAdministrator

Shows the complete current server profile in private paginated responses. Use it to audit the selected edition, configured channels, roles, guild identity and remaining setup work.

/install refreshAdministrator

Refreshes configured public panels without launching the heavier Tibia character scan. It is the safe maintenance action when panels need to be redrawn after a configuration change.

/guild setAdministrator

Registers, verifies, updates or removes the Tibia guild used by this Discord server. The verified guild also establishes the allowed world and becomes the source for rank, membership and registration checks.

/guild chatAdministrator

Assigns the Guild Chat channel used for Luminox's most important operational alerts, including significant enemy and blacklist outcomes. It does not replace the dedicated moderation or tracker logs.

/ranks mapAdministrator

Connects one Tibia guild rank to one Discord role. During synchronization, Luminox applies the mapped role that matches the member's verified main character.

/ranks guestAdministrator

Sets the Discord role used for registered characters who belong to the configured world but not the configured guild. This keeps legitimate guests separate from verified guild members.

/ranks overviewAdministrator

Displays every Tibia-rank mapping and the current Guest-role configuration. Run it before changing roles to verify which mappings are active or missing.

/moderators roles addAdministrator

Adds one Discord role to Luminox's global moderator-role list. Members with any configured moderator role gain the shared moderator access used by supported panels and workflows.

/moderators roles removeAdministrator

Removes one role from the global moderator-role list without deleting the Discord role itself. Specialist roles remain independent and may still be required by GuildBank, Support or Recruitment.

/moderators roles overviewAdministrator

Shows all roles currently recognized as global moderators. Use this private overview when checking why a staff member can or cannot access a moderation action.

/transfer accountAdministrator

Moves Luminox account data from one Discord account to another when a member changes accounts. Registrations and linked progress are transferred as one administrative operation rather than recreated manually.

Public control panels

Panel and channel configuration

/registration setAdministrator

Assigns the Character Registration channel and publishes its guided panel. Members then register, inspect or delete characters, change their main and configure timezone through buttons.

/registration refreshAdministrator

Redraws the existing Registration panel in its configured channel without changing registration data. Use it after a panel was manually removed or its presentation needs to be restored.

/guildlist setAdministrator

Assigns the channel for the full automatic guild roster. The panel groups members by guild rank and uses verified cached character data for levels, registration state and Guild Power.

/leaderboards setAdministrator

Assigns the combined Leaderboards channel and publishes its clickable index. Luminox manages the individual rankings, pagination and panel order automatically.

/levels setAdministrator

Assigns the channel for guild level-up announcements and rolling level-gain summaries. The summary panel stays below individual announcements and reuses already collected character data.

/deaths setAdministrator

Assigns the channel for verified guild death announcements and rolling death totals. Duplicate or historical deaths are filtered before a new announcement is published.

/watchlist setAdministrator

Assigns either Guild Members Online or Enemies Online to a channel. The selected panel refreshes from cached online data and can maintain its channel counter automatically.

/watchlist overviewAdministrator

Shows the current watchlist channels and stored configuration without modifying anything. Manual enemy management is intentionally handled through the linked Blacklist panel instead.

/finder setAdministrator

Publishes Finder in the selected channel. The panel combines cached online characters into compatible teams and exposes Looking for team and Match history controls.

/guards setAdministrator

Publishes the Guards panel and assigns the additional alert role. Eligible guild members can activate guard duty or report a blacklisted character from that panel.

/guildhall setAdministrator

Publishes the Guildhall panel and stores its additional specialist role. Administrators always manage it; moderators require any global Moderator role plus this Guildhall role.

/guildhall overviewAdministrator

Shows the configured channel and specialist role, current guildhall, declared room capacity, configured or occupied rooms and pending sale-report or room-claim totals without changing public data.

/guildhall refreshAdministrator

Redraws the existing Guildhall panel. Permanent audit embeds remain above it while the live controls return to the final channel position.

/events set huntsAdministrator

Assigns the Hunt Board channel. Hunt creation, vocation composition, shared-experience restrictions, signups and threads are then managed through the board buttons.

/events set bossesAdministrator

Assigns the Boss Board channel. Boss events prioritize tanks, healers and damage roles while preserving signup order and minimum-participant rules.

/events set questsAdministrator

Assigns the Quest Board channel. Quest events accept any vocation, support recruiting without an initial date and can schedule later through the event workflow.

/events set generalAdministrator

Assigns the general Event Board channel for unrestricted guild activities. The creator chooses the minimum attendance and whether scheduling begins immediately or after recruitment.

/loot setAdministrator

Publishes the Loot Split panel in the selected channel. Members paste Tibia session data and complete the guided split flow without using the retired member-facing loot command.

/autodelete setAdministrator

Creates or edits cleanup for one text channel. Always deletes every new unpinned message immediately; Timed uses a retention period from ten minutes up to thirty days.

/autodelete disableAdministrator

Explicitly disables Autodelete in the selected channel. It removes either an Always or Timed rule without requiring its previous mode or duration, and safely reports when the channel was already disabled.

/moderation setAdministrator

Assigns the operational moderation-log channel. Character registrations, role changes, member departures and Blacklist or Ban List actions are recorded there as Discord embeds.

/moderators set votingAdministrator

Publishes the private Staff Voting panel in the selected channel. Only eligible staff can open or vote on internal candidates, while administrators conclude each vote.

/moderators set applicationsAdministrator

Publishes the public Staff Applications panel. Eligible guild members can submit structured applications and the resulting candidacies remain visible for community voting.

Economy and progression

GuildBank, Loyalty and Recruitment

/guildbank set logAdministrator

Assigns the channel containing permanent GuildBank entries and the review control panel. Pending operations, confirmed balance and staff review controls are kept together there.

/guildbank set treasurerAdministrator

Sets the additional specialist role required for moderator transaction review. Administrators retain review access automatically, while moderators need both moderator and treasurer authorization.

/guildbank set minimumAdministrator

Configures the minimum accepted deposit and the accumulated amount required before staff review. Smaller eligible deposits remain attached to the account until the review threshold is reached.

/guildbank overviewAdministrator

Shows private GuildBank configuration, review roles, thresholds and moderation status. It is intended for auditing settings, not for creating a transaction.

/guildbank refreshAdministrator

Reposts the GuildBank control panel at the bottom of its channel while preserving permanent log entries above it. Use it when the panel order was disrupted.

/loyalty setAdministrator

Assigns either the public Loyalty activity panel or the Server Boost log, depending on the selected panel type. Existing account balances and reward history are not reset.

/loyalty messagesAdministrator

Enables or disables message-based Loyalty rewards in one text channel. Commands and bot interactions do not count as normal member messages.

/loyalty streamsAdministrator

Enables or disables Stream Loyalty in one voice channel. Valid screen-share time is tracked from Discord voice events and rewarded under the configured minimum and rolling limit.

/loyalty-adjustAdministrator

Adds or subtracts Loyalty Points from one eligible account with a mandatory reason. The manual adjustment becomes a visible Loyalty history entry rather than silently changing the balance.

/recruitment setAdministrator

Publishes Recruitment Rewards and assigns its specialist review role. Claims, disputes, retention protection and Loyalty rewards are handled through the resulting panel.

Safety and staff workflows

Moderation, support and maintenance

/automod enableAdministrator

Enables Luminox link and restricted-word protection in one selected text channel. The setting applies only to that channel and does not rewrite its Discord permission overwrites.

/automod disableAdministrator

Removes one selected channel from Luminox Automod protection. Other protected channels and their configured rules remain active.

/automod overviewAdministrator

Shows every channel protected by Luminox Automod, the current restricted-word policy and the configured temporary isolation duration.

/automod add-wordAdministrator

Adds one prohibited word or short phrase to the shared Automod policy. Matching messages in protected channels can be removed even when users disguise spacing or punctuation.

/automod remove-wordAdministrator

Removes one existing restricted word or phrase through the configured selector without disabling link protection or other prohibited terms.

/automod isolationAdministrator

Sets the temporary Discord isolation duration applied after a prohibited-word violation. Use the overview afterward to verify the current moderation policy.

/blacklist setAdministrator

Assigns the Blacklist channel and publishes its live list plus administration controls. Character and guild records are added, edited or removed through panel buttons rather than separate CRUD commands.

/blacklist overviewAdministrator

Shows Blacklist configuration and stored entries privately without refreshing Tibia data. Use it to verify the assigned channel and moderation state.

/banlist setAdministrator

Assigns the Ban List channel and publishes its records and staff controls. Entries can represent registered accounts or external Tibia characters and include reason, duration and moderator attribution.

/banlist overviewAdministrator

Displays private Ban List configuration and current records. It is useful for checking an active or unlimited ban without changing the public panel.

/tracker setAdministrator

Assigns the Character Tracker channel for permanent identity-change logs and the current summary panel. The tracker records renames, trades, transfers, returns and removals affecting monitored characters.

/support setAdministrator

Publishes the private Support Ticket panel in the selected clean text channel. Ticket conversations themselves live in private Discord threads rather than cluttering the panel channel.

/support categoryAdministrator

Creates a new support category with its description, required user role, questions and up to five specialist staff roles. The global moderator or administrator requirement still applies to staff access.

/support edit-categoryAdministrator

Opens an existing category through autocomplete and allows every category detail to be revised. Use it to change questions, visibility or specialist roles without deleting ticket history.

/support remove-categoryAdministrator

Removes one selected category from the ticket-opening panel. Existing archived Discord threads remain governed by Discord and are not rewritten as part of the category removal.

/support overviewAdministrator

Shows private support configuration, every category and the staff roles assigned to each one. The response remains visible long enough for administrators to review multi-page settings.

/support refreshAdministrator

Forces the Support panel to be redrawn in its configured channel. It does not close, reopen or modify existing ticket threads.

/website setAdministrator

Publishes the Website panel in a clean text channel and stores its public HTTPS address. The panel monitors availability and opens private media-proposal threads without storing uploaded image bytes in the bot database.

/website overviewAdministrator

Shows Website health, panel location, proposal counts and whether the private publishing connection is ready. It reports missing setup by name but never exposes credential values.

/website refreshAdministrator

Rechecks the public HTTPS website and edits the existing Website panel in place. It does not create another panel message or republish any media.

/website publishAdministrator

Works only inside the matching approved private Website submission thread. It sends that approved proposal to the fixed publishing process; no image is downloaded onto the Luminox host computer.

/update-allAdministrator

Runs the full registered-character synchronization for the current Discord server. Because it performs the heavier refresh path, use it for deliberate recovery or auditing rather than routine panel updates.

Panel handlers are intentionally absent from Discord's command list

/register, member-facing /loot, direct event creation and Blacklist or Ban List CRUD commands were replaced by guided panels. Internal modules still process their buttons, selectors and modals, but members see one clear entry point instead of duplicated slash commands.
