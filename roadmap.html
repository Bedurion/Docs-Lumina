Roadmap

Build from real use.

Founder Edition remains Luminox's live proving ground inside Lumina. Community and Universal capability profiles already exist; the roadmap now focuses on reliable public delivery, subscriptions, support and safe growth.

Delivery path

From Founder proof to reliable public use.

Every stage has a clear purpose: test the feature in Lumina, prepare it safely and release it publicly only when it remains dependable.

Now

Founder hardening

Operate the complete system under real Lumina activity and remove edge cases before they reach public guilds.

Built

Public edition foundation

Community and Universal already define separate feature access, Free and Premium tiers, limits, refresh policy and retention.

Next

Reliable public release

Connect subscription delivery, verify upgrades and downgrades, strengthen deployment and make every first installation understandable.

Later

Validated expansion

Adapt only the Founder features that prove useful and safe for the relevant public edition instead of copying Tibia assumptions everywhere.

Release principles

What determines whether a feature moves forward.

Real use first

A feature should solve a recurring guild problem, not exist only because another bot has it.

Safe upgrades

New configuration must detect existing data and avoid destructive setup or duplicate panels.

Sustainable data

Temporary information expires, old detail follows clear limits and complete Discord conversations stay in Discord.

Clear permissions

Every member, moderator, specialist and administrator action needs an explicit access model.

Measured performance

Information already collected is reused, while heavier game checks remain separate from simple panel updates.

Documented delivery

A system is not finished until setup, daily use, edge cases and recovery are understandable.
