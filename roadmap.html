Roadmap

Build depth first. Package reliability second. Scale only when the foundation is ready.

Luminox develops inside Founder Edition, where real Lumina workflows expose edge cases before systems are prepared for Community and Universal guilds.

Now

Founder Edition depth

Complete and harden identity, events, Loyalty, economy, support, recruitment, staff systems and intelligence under real guild use.

Next

Public-product boundaries

Separate Founder-only cadence from Community defaults, document limits, finalize retention rules and make every setup path safe on a new server.

Then

Community readiness

Package Tibia-specific systems for other guilds, add deployment and support operations, define Free/Premium differences and validate upgrade behavior.

Later

Universal Edition

Extract game-agnostic events, support, staff, economy and automation concepts without forcing Tibia terminology into other communities.

Release principles

What determines whether a feature moves forward.

Real use first

A feature should solve a recurring guild problem, not exist only because another bot has it.

Safe upgrades

New configuration must detect existing data and avoid destructive setup or duplicate panels.

Sustainable data

Temporary state expires, detailed history is bounded and Discord-native history stays in Discord.

Clear permissions

Every member, moderator, specialist and administrator action needs an explicit access model.

Measured performance

Cached data is reused and expensive game scans remain distinct from lightweight panel refreshes.

Documented delivery

A system is not finished until setup, daily use, edge cases and recovery are understandable.
