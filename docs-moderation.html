Scope

Automod and audit are edition-independent

Both public editions can protect selected channels and publish a moderation log. These systems use Discord identity and permissions, so they do not depend on Tibia characters or guild data.

Separate list guides

Use Ban List documentation for timed or unlimited sanctions. Use Blacklist documentation for Community Edition Tibia intelligence.

Channels

Configure protection and audit separately

Moderation log/moderation set channel:#log
Enable one Automod channel/automod enable channel:#general
Inspect one channel/automod overview channel:#general
Disable one channel/automod disable channel:#general

Automod can be enabled in multiple channels up to the current plan limit. Rules are stored per channel, not globally assumed for the whole server.

Channel protection

Detect links even when the text tries to hide them

Luminox normalizes common separators, whitespace and obfuscation patterns before deciding whether a message contains a link. Enforcement applies only to configured channels and only when the author lacks the Discord permission that allows the content.

  • Normal clickable links are detected.
  • Plain-text domains and common split-link tricks are evaluated.
  • Messages outside enabled channels are not affected.
  • Authorized staff or members with the required Discord permission are exempt.
  • The removed message is not copied into an additional Luminox conversation database.

Blocked language

Extend the default rules per channel

Add a word/automod add-word channel:#general word:example
Remove a word/automod remove-word channel:#general word:example
Review current words/automod overview channel:#general

Use specific words or short expressions that represent a real moderation rule. Avoid overly broad fragments that could match normal conversation accidentally.

Temporary isolation

Choose the consequence explicitly

/automod isolation channel:#general

The configured isolation period determines how long a violating member is temporarily restricted after enforcement. Review the overview after changing it and test with a normal member account.

Discord hierarchy still applies

Luminox can only moderate members below its highest Discord role and only when the required permissions are granted.

Audit trail

Important changes remain as fixed Discord messages

The moderation log can record character registration, managed role transitions, detected guild departures and staff actions from connected systems. Each entry identifies the relevant member or target, the action, responsible account when applicable and recorded time.

Historical messages remain fixed in Discord. Live panels refresh separately, so current state does not overwrite the audit trail.
