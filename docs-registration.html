Member flow

First registration is one guided process

  1. Click Register character and enter the exact Tibia name.
  2. If no timezone exists, choose it inside the same flow.
  3. Copy the temporary LUMINOX-… code.
  4. Place it at the beginning of the character comment on Tibia.com.
  5. Click Register character again and submit the same name.
  6. After success, remove the code from the Tibia comment.

The timezone remains saved even if verification needs another attempt. The temporary registration response remains available longer than ordinary informational responses so the code can be copied safely.

Validation

What can and cannot register

  • The character must exist on Tibia.com.
  • The character must belong to the Discord server's configured world.
  • The character does not need to belong to the configured guild; outside characters become eligible for Guest treatment instead.
  • An account can hold up to ten registered characters.
  • A character already owned by another Discord account cannot be duplicated.
  • Blacklist and Ban List restrictions are explained explicitly and prevent normal guild access.
World boundary

If the server is configured for Secura, a character from Antica cannot register even when the name is valid.

Main character

The account's public identity

The first registered character becomes main automatically. Use Change main character when more than one exists. If the main leaves the guild but another registered character remains inside, Luminox can promote the highest-level eligible character automatically.

Discord synchronization

Nickname and role follow the eligible main

NicknameName [RP360 L8]: vocation, current level and account Loyalty Level.
Guild roleMapped from the Tibia guild rank of the eligible main.
GuestUsed when characters are registered but none belongs to the configured guild.
No registrationsLuminox removes its managed guild roles and clears the managed nickname.

Discord does not allow bots to modify the server owner's nickname or members whose highest role is above the bot.

Character management

Everything stays inside the panel

  • My characters shows every registration and current main.
  • Change main character opens a selector.
  • Delete character removes a selected registration after confirmation.
  • Set timezone remains available independently.
  • /update refreshes only your current main, nickname and role.
