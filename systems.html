Documentation · How it works

Collect useful information once. Reuse it across the guild.

Luminox avoids repeated game checks by sharing current data between compatible panels. This keeps updates fast without asking the same external service for the same answer again and again.

Simple flow

Member action, system check, useful Discord result

1. A person acts

They use a command, panel, button, selector or form.

2. Luminox applies the rules

The relevant character, event, Loyalty, bank or staff system checks the request.

3. Shared information is reused

Guild settings, registered identities and current online data are read once where possible.

4. Discord shows the result

The bot updates a panel, counter, thread, log or private response.

Shared data

One online snapshot powers multiple features

Members Online, Enemies Online, Finder, playtime, event attendance and session statistics reuse compatible cached data. A panel refresh does not imply a new request for every displayed character.

Result

Fast Founder Edition panels without multiplying Tibia requests as new visualizations are added.

Storage

Keep important totals; clean temporary detail

  • Account balances remain even after old reward details are cleaned.
  • Looking for team, Guard duty and active reports remove themselves after their useful period.
  • Support and event conversations remain in archived Discord threads.
  • GuildBank and recruitment keep enough information to review or reverse a reward correctly.
  • Guildhall keeps current rooms and occupants while permanent decisions remain visible in Discord logs.
  • Daily backups retain 30 copies; monthly backups retain 12.

Editions
