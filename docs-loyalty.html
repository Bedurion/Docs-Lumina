Panel setup

Publish Loyalty and Boost panels separately

Use the same command with a different system choice. Both panels can live in different channels.

Loyalty panel/loyalty set channel:#loyalty system:Loyalty Points panel
Server Boost log/loyalty set channel:#boosts system:Server Boost log
Message rewards/loyalty messages channel:#channel enabled:true. Commands and very short messages do not count.
Stream rewards/loyalty streams channel:Voice enabled:true. Only screen-share time in enabled voice channels is eligible.
Manual correction/loyalty-adjust user:@member points:number reason:text is administrator-only and creates history.

Eligibility

Who can earn and keep Loyalty Points

  • The Discord account needs at least one verified character.
  • At least one registered character must belong to the configured guild and world.
  • The account cannot be actively restricted through the Blacklist or Ban List.
  • Leaving the Discord server, losing every guild character or becoming restricted resets the account balance.
  • Changing the main character does not move or duplicate points because the balance belongs to the account.
Registration is not enough by itself

A registered guest outside the configured guild can use allowed guest features, but cannot accumulate guild Loyalty Points.

Personal profile

What View my Loyalty profile shows

  • Discord account link and current balance.
  • Current Loyalty Level, next threshold and points remaining.
  • Leaderboard position and GuildBank contribution.
  • Events created and successful events.
  • Active boosts, completed boost periods and pending boost rewards.
  • Total valid stream time, rewarded time, earned points, pending points and rolling usage.
  • Paginated recent reward history.

History and sustainability

The balance stays; old reward detail is cleaned

The total balance remains while the account is eligible. Individual reward entries follow age and maximum-count rules so the data store does not grow forever. Public panels show the most useful recent rewards and automatically split into multiple messages when necessary.
