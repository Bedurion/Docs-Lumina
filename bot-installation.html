The template and the bot perform different jobs.

Discord creates the starting server layout. Luminox is authorized separately and then connects its systems to that layout. The bot never silently recreates or deletes your server structure.

Before you install

Clear answers for the first decision.

The technical guide contains exact commands. These answers explain the installation model before you authorize the bot.

Does Luminox create or delete my Discord channels automatically?

No. The guided installer audits the current structure and explains what each selected function needs. You deliberately choose or create the channels and roles that Luminox will use.

Is the template route required?

No. It is the recommended starting point for new servers because it makes the final organization easier to visualize. Existing communities can preserve their current layout and connect Luminox to it.

Do members need to use slash commands?

Daily workflows are panel-first. Administrators use setup commands to configure and publish systems; members normally use buttons, selectors, guided forms and private responses.

Can I begin with only one function?

Yes. A modular rollout is encouraged. Configure one valuable workflow, test it with real permissions and expand only when the community is ready.

Can I change edition later?

Administrators can review edition and configuration choices without treating the server as disposable. Availability and migration checks still apply to edition-specific data and functions.

Do I need Premium before installation?

No. Universal and Community both start with a Free plan. Premium becomes relevant only when your community needs additional functions, capacity, history or supported refresh speed.
