Documentation · Website Publishing

Review community media in Discord and publish without touching the bot computer.

The Website panel turns Discord into a controlled proposal and review interface. Original attachments remain in private threads; an approved publication is downloaded, validated and re-encoded only inside an ephemeral GitHub Actions runner.

Member flow

One private proposal thread per submission

  1. A member presses Submit content in the Website panel.
  2. The modal requires a public title, description and accessible alternative text; credit is optional.
  3. Luminox creates a private thread containing only the author and eligible staff.
  4. The author uploads exactly one JPG, PNG or MP4 file, up to 10 MB.
  5. Ready for review validates Discord attachment metadata and locks the exact attachment ID into the proposal.

Submitting confirms that Lumina has permission to publish the media and its chosen credit. Unsupported files, direct URLs, WebP, SVG, GIF, PDF, archives and executables are rejected before review.

Staff review

Moderators review; administrators publish

Any configured global Moderator role or Discord administrator can approve, reject or request changes. Rejections and requested changes require an explanation. Approval does not publish immediately: it creates a second security boundary where only a Discord administrator can start the GitHub workflow.

Final publication

Use the Publish button or /website publish

The command works only inside the corresponding approved private thread. Luminox refreshes the stored Discord attachment ID to obtain its current signed CDN address, encrypts the complete payload with AES-256-GCM and triggers the fixed GitHub workflow. The bot never downloads or decodes the media.

A read-only GitHub job checks the Discord CDN hostname, HTTPS path, MIME type, declared size and binary signature. JPG and PNG files are decoded with a hard pixel limit, resized when needed, stripped of embedded metadata and converted to WebP. MP4 files must use H.264 video with optional AAC audio, last no more than 120 seconds and are re-encoded with metadata, chapters, subtitles and data streams removed. A separate write-enabled job rejects unexpected paths, symbolic links, altered history, malformed metadata or invalid output before committing. The static gallery uses DOM text nodes rather than interpreting submitted text as HTML.

Security

Bounded inputs and minimal credentials

  • Exactly one JPG, PNG or MP4 file, up to 10 MB.
  • MP4 videos are limited to 120 seconds and H.264 video with optional AAC audio.
  • Declared MIME type and binary signature must match.
  • No original filename becomes a website path.
  • No attachment bytes are stored in the Luminox JSON database.
  • Signed Discord attachment URLs are encrypted before entering GitHub workflow inputs.
  • Media parsing runs without repository write permission.
  • The write-enabled job accepts only a strictly validated isolated artifact.
  • The workflow is serialized to prevent simultaneous gallery-data writes.
  • GitHub Actions uses a locked dependency graph and official actions pinned to commit hashes.
  • The bot token must be a fine-grained credential limited to Actions write access on this documentation repository only.
  • Published, rejected and cancelled metadata is bounded per Discord server.
Never paste credentials into Discord

GitHub tokens and the shared payload key belong only in the bot .env and GitHub Actions secrets. Luminox intentionally never stores or displays them in panel configuration.

Administrator setup

Configure secrets first, then publish the panel

/website set channel:#website url:https://example.github.io/site/website overview/website refresh

The bot environment requires the public website address, authorized Founder server, GitHub owner, repository, workflow file, branch, fine-grained token and a 32-byte payload secret. Add the same payload secret to the GitHub repository as WEBSITE_PUBLISH_PAYLOAD_SECRET. In GitHub repository settings, set the Pages source to GitHub Actions so the final least-privilege deployment job can publish the updated static site.

Generate the shared key locally with openssl rand -base64 32. Keep the value private, restart Luminox after updating its environment and use /website overview to confirm that the isolated publisher reports Ready.

Discord permissions

Panel channel and private threads

  • View Channel, Send Messages and Embed Links.
  • Read Message History.
  • Create Private Threads and Send Messages in Threads.
  • Manage Threads so Luminox can add reviewers, lock and archive completed proposals.
  • Manage Messages if the panel channel is protected against normal chat.

The panel message is edited in place during health checks and queue changes. Routine refreshes do not repost the panel or generate new-message notifications.
