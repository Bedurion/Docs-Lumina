Documentation · Timezones

Write local time; let Discord display local time.

Luminox stores IANA timezones, interprets event dates in the creator's zone and publishes Discord timestamps for every viewer.

Personal command

Configure once

/timezone set zone:Europe/Madrid

Use a standard IANA timezone such as America/New_York, Europe/Warsaw or Asia/Tokyo. This setting belongs to the Discord account.

Registration

The first character guides missing timezone setup

If no timezone exists, the registration panel asks for it inside the same guided flow before verification continues. Once stored, failed or repeated registrations do not ask again.

Event scheduling

Creator input, viewer output

  1. The creator enters a date in their configured timezone.
  2. Luminox converts it to one absolute instant.
  3. The event panel uses Discord timestamps.
  4. Each viewer sees the same instant rendered in their local Discord client.

Event creation requires a configured timezone to prevent silent server-time assumptions.

Message conversion

Apps → Convert time

Use the message context command on a message containing a time. Luminox compares the author's timezone with yours and posts a short conversion message that is automatically removed. Clock-reaction compatibility may also be available where enabled.
