Panel setup

Channel and alert role are both required

/guards set channel:#guards role:@Alerts

The panel publishes Enable alerts, Disable alerts and Report enemy. The channel title shows the number of active guards, while the panel identifies each selected guard character and current active duration.

Eligibility

Guard duty represents a verified guild member

  • The Discord account needs a registered main character.
  • The selected character must belong to the configured guild and world.
  • A Guest or restricted account cannot activate guild Guard duty.
  • The Alerts role is additional; it does not replace the member's guild rank or unrelated Discord roles.
  • Trying to enable or disable the current state returns a clear private response without repeating role operations.

Guard duty

Registered guild mains only

  1. Click Enable alerts.
  2. Select an eligible registered guild character or save the main character as default.
  3. Luminox grants the configured alert role without changing rank roles.
  4. The public panel shows the chosen character and active duration.

Guard duty automatically expires after five hours. Repeated enable/disable actions are protected by a dedicated account-wide cooldown.

Enemy report

Targets come from the Blacklist

The reporter chooses a blacklisted character, selects their own reporting character and writes the location. Arbitrary names cannot be submitted, which keeps alerts connected to existing bounty and reason data.

Report content

Enemy name, vocation and level, active duration, Tibia Coins, blacklist reason and reporting character. The report mentions the configured Alerts role once so opted-in guards receive the operational notification.

If a character changed name, the Blacklist and Tracker identity history remains the source. Guards does not create a detached duplicate record under the former name.

Coordination

Every report opens a temporary thread

Luminox creates a thread named with the enemy character and current vocation/level so guards can coordinate without filling the panel channel. New thread activity extends the inactivity window.

Use the thread for movement updates and coordination. The report card remains the main source for location, reporter, bounty and Blacklist reason, so that information does not need to be repeated in every message.

Abuse protection

Separate cooldowns

Guard role toggles and enemy reports use independent cooldowns, so reporting cannot be spammed by repeatedly changing duty state or switching registered characters.

Cleanup

Two hours of inactivity

  • Reports expire after two inactive hours.
  • The creator can request early deletion.
  • Early deletion records who removed it, locks the thread and keeps the notice visible for ten minutes before cleanup.
  • Expired temporary report data is removed instead of accumulating permanently.
  • Guard duty itself expires after five hours even when no report was created.

Connected moderation

Guards reports; Blacklist defines the target

Use the Blacklist panel to add, edit or remove sanctioned characters and Guilds. Use Tracker to follow renames, trades, transfers and returns. Guards consumes that maintained identity and reason data for short-lived field reports.
